Catalog of 190+ vector databases and RAG infrastructure tools — embeddings stores, retrieval engines, RAG frameworks, hybrid-search backends.

What you'd use this for

  1. Vector DB selection — "Pinecone vs Qdrant vs Weaviate vs Milvus at our scale" — filter category=vector-db-engine + sort by GitHub stars + filter by deployment-type (cloud / self-hosted) + scale tier. The dataset captures the 52% of records that ARE vector DB engines (118 of 225) so you don't sift through RAG frameworks too.
  2. RAG framework comparison — Building a new RAG pipeline. Filter category=rag-framework → see langchain RAG / llama_index / haystack / langroid / ragatouille all in one view. Compare features: streaming, source citation, multi-modal, reranking-built-in.
  3. Reranker shopping — RAG quality is plateauing. Filter category=reranker → cross-encoder vs Cohere reranker vs ColBERT vs BGE reranker. Each takes a different speed/quality trade-off — the dataset surfaces them all.
  4. Embedding library audit — Need to switch from OpenAI embeddings to self-hosted. Filter category=embedding-library + category=embedding-model → sentence-transformers / FastEmbed / Voyage / BGE / Nomic / E5. Cross-reference with huggingface-models (4,000 records) for the underlying open-weight models.
  5. RAG evaluation tooling — "How do I measure my RAG quality?" Filter category=rag-eval → 38 records: RAGAS / DeepEval / TruLens / continous-eval / BEIR benchmarks. Pick by RAG-specific eval support.
  6. Hybrid search architecture — Building production search. Filter category=hybrid-search for BM25 + dense combinations. Filter category=search-engine for Elasticsearch / OpenSearch / Meilisearch / Typesense alternatives. The two together cover the "search infrastructure" decision.
  7. Cross-sell ML 6-pack — Pair with ai-models-pricing (hosted APIs) + huggingface-models (open weights) + huggingface-datasets (training data) + ai-prompts (prompts) + llmops-and-eval (platform tooling) → complete LLM/ML engineering stack. The dataset's role: the retrieval layer that connects user query to model context.
