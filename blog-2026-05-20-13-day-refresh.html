What the numbers say

npm is the loudest signal

The npm-packages dataset (top-N-by-downloads ranking) had a 122% gross churn in 13 days — 3,760 packages broke into the top-6,000 and 3,775 dropped out, leaving the net total roughly flat. This is what an actively-ranked feed looks like: a static "top 6,000 npm packages" snapshot taken today and one taken two weeks ago disagree about more than half of their members. If you're building any kind of "popular libraries" UI, you cannot serve a stale snapshot.

HuggingFace datasets churn is heavy too

The HuggingFace datasets feed had 43.3% gross churn: 682 new dataset uploads and 539 deletions across 13 days. HuggingFace Hub is publishing-active in a way that GitHub awesome-lists are not.

Awesome-list-backed datasets are mostly idle

Everything below 5% churn (14 of 20 datasets) is sourced from one or more awesome-* README files maintained by humans on GitHub. These move at PR-cadence: a few entries added per week, occasionally a stale link removed. llmops-and-eval, vector-db-and-rag, and design-resources had literal zero adds or removes in 13 days — the lists are not actively curated, only the metadata (descriptions, GitHub stars, etc.) refreshes when we re-crawl.

Implication for downstream users

If you're picking which of our datasets to subscribe to (or whether to bother subscribing at all vs taking the snapshot once), the table above is the actual refresh-value distribution. npm and the HF feeds justify high-frequency pulls. The awesome-list-backed ones do not. A monthly refresh is plenty for design-resources or vector-db-and-rag; nightly or weekly is correct for npm or HuggingFace.
