Full list, programmatic access

The full 208-entry list is served as JSON at /cross-dataset-overlap.json (schema: CrossDatasetOverlap in OpenAPI 1.3.0). Recomputed on every catalog refresh from the live liveRecordCount data files. Worked jq snippets at examples.html#cross-leverage:

curl -s https://futdevpro.github.io/niche-datasets-free/cross-dataset-overlap.json \
  | jq '.topByCount[] | select(.count >= 4) | .url'

How the count is computed

Each entry in every catalog dataset has a canonical url field. We aggregate URLs case-insensitively, group by dataset slug, and emit any URL that appears in 3+ distinct slugs. Threshold of 3 keeps the output to ~200 entries (vs ~1,300 at 2+, which is dominated by near-duplicates and not very useful). Methodology in buyer's guide #cross-leverage and FAQ Q28.

The crawler also strips two common upstream noise patterns at extraction time, so the cross-niche counts reflect real tools — not section-headers leaked into the entry stream. See api.html#data-quality for details and examples.html#noise-audit for a buyer-runnable verification jq probe.
