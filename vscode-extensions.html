Catalog of 4,800+ VS Code Marketplace extensions with install counts, version, last-updated, extension ID, and publisher.

What you'd use this for

  1. Internal "approved extensions" catalog for an engineering team — A 100-person dev team needs a curated default extension list per language. Filter category=language-syntax + sort by installCount → top extensions for Python, TypeScript, Go, Rust. Ship the list as a .vscode/extensions.json recommendation file. New hires get the right setup on day 1.
  2. Pair-programming setup baseline — Recommend a Devcontainer / Codespaces config with the top general-purpose extensions: Prettier (68M), Live Server (78M), GitHub Copilot (73M), GitLens. The dataset's installCount shows what your future colleagues are actually using.
  3. Plugin-ecosystem competitive analysis — You're building a JetBrains alternative. Filter the dataset by category → see VS Code's category-share-of-voice vs your own marketplace. The 46% language-syntax dominance shows where users invest the most plugin time.
  4. AI-assistant market research — Filter category=ai-assistant (60+ extensions). Top 2 (Copilot + Copilot Chat) hold 146M of the installs. Tabnine, Codeium, Continue, Cursor (extension version) are the contenders for the long tail. Useful for "what AI coding tools is the market actually buying" research.
  5. Theme-pack / icon-pack sourcing for a sister product — Building a code-editor app. The themes-and-icons category (12% of top-4800) shows what color schemes / icon packs developers actually install. Top: Dracula, Material Theme, One Dark Pro, Tokyo Night. License each + ship as defaults.
  6. Newsletter / "extension of the week" content series — Filter by category + sort by lastUpdated → recently-updated extensions in a specific category make for fresh content. "Best new Python extensions you missed in Q1" — sourced from this filter.
  7. Engineering platform team's onboarding kit — Platform team building a "new engineer setup" script. Extracts top-installed extensions from this dataset by language / category → generates a personalized VS Code config based on the engineer's role (backend / frontend / data / etc.). Kit gets engineering productive in hours, not days.
