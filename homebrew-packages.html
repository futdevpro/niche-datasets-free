Homebrew Packages Directory

12,200+ records · Free sample (20 records, JSON + CSV) · Full dataset $11 · Sample refreshed 2026-05-13

Catalog of 12,200+ Homebrew formulae and casks with install counts, package type, dependencies, tap, and deprecation status.

What you'd use this for

  1. Dev-environment bootstrap script — You're hiring devs and need a bootstrap.sh that installs the canonical macOS dev stack. Filter dataset for top-N CLI tools by install count → script becomes "brew install gh awscli node python ..." with the most-vetted toolset, no guesswork.
  2. Devcontainer / Dockerfile base-image generator — You're building a Devcontainer image. Filter dataset by category=cli-tool + sort by installCount365d → top-N system tools that should be in your base image. Cross-reference with npm-packages for project deps and vscode-extensions for IDE plugins.
  3. Internal "approved packages" catalog for an enterprise — Compliance team needs an allowlist. Filter dataset by license=apache-2.0|mit|bsd-* (64% have license info) → ship as a brew-allowlist file. Engineers see "available packages" instead of free-form brew install.
  4. Deprecation audit for an existing dev environment — Run brew list on a long-running dev machine. Cross-reference each package's name against this dataset's deprecated=true filter (10% of catalog) → flag everything that's been deprecated. Replace with the dataset's deprecation_replacement_formula field. Saves a half-day audit.
  5. CLI-tool discovery / "what should I be using?" content — Newsletter or blog post: "Top 10 CLI tools you didn't know about". Filter dataset by installCount365d > 100K (vetted) AND name not matching common-knowledge tools (git, curl, etc.) → underrated tools surface with their stats. Each becomes a tutorial.
  6. Cask / desktop-app catalog for a new Mac — Filter packageType=cask + sort by install count → top desktop apps brew-distributable: Slack, Zoom, VS Code, Chrome, Discord, etc. Build a "new Mac setup" script that brew-installs them in one command.
  7. AI-tooling adoption tracker for the macOS dev community — The brew package universe is starting to absorb AI tooling: ollama, llama.cpp, mlx-related packages, AI CLIs. Filter dataset by name match for AI keywords → trend cohort showing where Apple Silicon AI infrastructure is heading. Useful for AI-tooling startups picking distribution channels.
