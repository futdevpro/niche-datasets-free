API — Free Sample Catalog (OpenAPI 3.1)

3 read-only endpoints. No auth, no signup, no rate limit (Pages CDN bounds apply). Suitable for agents that need to enumerate the catalog programmatically.

Machine-readable spec: /openapi.json (OpenAPI 3.1). Drop it into openapi-generator or your favourite client codegen to get a typed SDK in your language.

Endpoints

GET /datasets.json

Enumerate all 20 datasets. Returns the full catalog as a single JSON document: name, description, license, homepage + the 20 dataset entries with slug, recordCount, refresh date, sample/full URLs, related-slug list.

Safe to poll daily. ~21 KB.

curl -s https://futdevpro.github.io/niche-datasets-free/datasets.json | jq '.datasets | length'
# → 20

GET /{slug}-meta.json

Metadata for one dataset. Returns a single dataset's metadata object — same shape as the entries inside /datasets.json but addressable on its own URL.

Useful for thin clients that only care about one or two datasets.

curl -s https://futdevpro.github.io/niche-datasets-free/ai-models-pricing-meta.json | jq .recordCount
# → "800+"

Valid slugs (20): ai-agents, ai-models-pricing, ai-prompts, ai-tools, cybersecurity-tools, design-resources, developer-tools, homebrew-packages, huggingface-datasets, huggingface-models, llmops-and-eval, mcp-servers, no-code-lowcode, npm-packages, open-source-alternatives, platform-engineering, public-apis, self-hosted-software, vector-db-and-rag, vscode-extensions.

GET /feed.xml

RSS 2.0 refresh feed. One entry per dataset, ordered by lastRefreshed. Subscribe in your feed reader to be notified when any dataset re-pulls.

Caching, freshness, rate limit

ConcernBehavior
CacheGitHub Pages CDN; default Cache-Control: max-age=600. Responses are freshness-stamped via the embedded lastRefreshed field — trust that over the HTTP cache header.
Rate limitNone on Pages itself. GitHub may rate-limit raw.githubusercontent.com on the sample URLs at extreme volume (~10k req/min from one IP) — back off with jitter.
Refresh cadencePer-source. The fastest-moving (npm, HuggingFace) get nightly-to-weekly pulls. The slowest (awesome-list mirrors) get monthly. See the 3-day update (2026-05-23) and 13-day refresh report for the per-source churn numbers that drive this cadence.
AuthNone on the sample tier. Full data ZIPs are gated on Gumroad purchase.

Two record-count fields — when to use which

Every dataset entry ships with both recordCount and liveRecordCount:

  • recordCount — string with round-floor and + suffix (e.g. "6,100+"). Marketing format, stable across small refresh diffs. Use for human-facing UIs.
  • liveRecordCount — integer with the exact count from the live data file (e.g. 6171). Use for size estimation, pagination, change-detection. May be omitted if the live file is unreachable at build time — treat absence as "unknown".
curl -s https://futdevpro.github.io/niche-datasets-free/datasets.json \
  | jq '.datasets[] | {slug, recordCount, liveRecordCount}'
# {"slug": "npm-packages", "recordCount": "6,100+", "liveRecordCount": 6171}
# {"slug": "homebrew-packages", "recordCount": "12,200+", "liveRecordCount": 12239}
# ...

Schema versioning

Every JSON response includes a schemaVersion field. Breaking changes will bump it; additive changes will not. Clients should ignore unknown fields (forward-compat). The OpenAPI spec itself has a separate info.version that follows semver — currently 1.2.0 (additive changes since 1.0.0: liveRecordCount + bundles in DatasetMeta).

Bundles graph — find which bundles include a dataset

Every /datasets.json entry and /{slug}-meta.json response ships a bundles array with {name, url} objects for each bundle that includes the dataset. Same data as the schema.org isPartOf in the JSON-LD, exposed as flat JSON for non-JSON-LD consumers (typed clients, simple curl-jq scripts, agents).

curl -s https://futdevpro.github.io/niche-datasets-free/npm-packages-meta.json \
  | jq '.bundles[] | {name, url}'
# {"name": "Complete Developer Data Bundle", "url": "https://jhonnyronnie.gumroad.com/l/developer-data-bundle"}
# {"name": "Platform Builder Pack",          "url": "https://jhonnyronnie.gumroad.com/l/platform-builder-pack"}
# {"name": "Dev Stack Pack",                 "url": "https://jhonnyronnie.gumroad.com/l/dev-stack-pack"}

Useful for buyer-agents that compare per-dataset price vs. bundle price programmatically, or for affiliate / referral systems that need to know which Gumroad SKUs to surface for a given dataset.

Cross-dataset URL overlap — high-cross-leverage tools

The endpoint /cross-dataset-overlap.json lists every URL that appears in 3+ of the 20 catalog datasets — the high-cross-leverage tools that span developer niches. Currently 208 URLs above the threshold; the top entries (5 datasets) include ClickUp (developer-tools + homebrew + no-code-lowcode + platform-engineering + public-apis), Snyk (ai-agents + cybersecurity + developer-tools + homebrew + public-apis), and KubeStellar Console (ai-agents + ai-tools + llmops-and-eval + mcp-servers + platform-engineering).

curl -s https://futdevpro.github.io/niche-datasets-free/cross-dataset-overlap.json \
  | jq '.topByCount[] | select(.count >= 4) | {url, count, appearsIn}'

Useful for market-mapping (which tools span the most niches?), cross-sell discovery (buyers of one dataset likely want adjacent ones if their target tool spans both), or for bundle-pricing decisions ("Qdrant appears in 4 of our datasets → justifies the ML Builder Pack").

Data quality — crawler-level noise filters

Awesome-list READMEs sometimes leak section-header structure into the entry stream. Two extractor-level filters strip the patterns at ingest, before normalization and dedupe see the records:

  • Year-prefix section-header: ^(19|20)\d{2}\s+[A-Z][a-z]+$ — e.g. "2021 Result", "2023 Competition". Typical for benchmark/conference references nested as list items under an h2.
  • Self-section pointer: bare github.com/<owner>/<repo>#anchor URL where the link name (alnum-normalized) equals the anchor (alnum-normalized). Typical when a README's table-of-contents leaks into the entry list.

Both filters live in shared/extract-markdown.ts isSkippable() and apply on every awesome-list refresh. Most recent sweep: 3 records removed from vector-db-and-rag, 27 from self-hosted-software, zero false positives across the remaining 18 datasets. Verify against any sample yourself with the noise-audit jq snippets.

Agent-callable example — Python with openapi-python-client

End-to-end: install the generator, point it at /openapi.json, get a typed client, enumerate the catalog from agent code.

# 1. Generate the client
pip install openapi-python-client
openapi-python-client generate \
  --url https://futdevpro.github.io/niche-datasets-free/openapi.json
cd niche-datasets-free-api-client
pip install -e .

# 2. Use it from agent / LLM-app code
from niche_datasets_free_api_client import Client
from niche_datasets_free_api_client.api.catalog import get_catalog
from niche_datasets_free_api_client.api.dataset import get_dataset_meta

client = Client(base_url="https://futdevpro.github.io/niche-datasets-free")

# enumerate all 20 datasets
catalog = get_catalog.sync(client=client)
print(f"{len(catalog.datasets)} datasets, last refreshed {catalog.last_refreshed}")

# fetch one dataset's metadata
meta = get_dataset_meta.sync(client=client, slug="ai-models-pricing")
print(f"{meta.name}: {meta.record_count} records, ${meta.full_dataset_price_usd}")
print(f"Free sample: {meta.sample_json_url}")

For a TypeScript agent runtime, swap the codegen for openapi-typescript (see Auto-generate a client above) and use the same flow with the typed fetch wrapper it produces.
