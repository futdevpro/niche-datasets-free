API — Free Sample Catalog (OpenAPI 3.1)

3 read-only endpoints. No auth, no signup, no rate limit (Pages CDN bounds apply). Suitable for agents that need to enumerate the catalog programmatically.

Machine-readable spec: /openapi.json (OpenAPI 3.1). Drop it into openapi-generator or your favourite client codegen to get a typed SDK in your language.

Endpoints

GET /datasets.json

Enumerate all 20 datasets. Returns the full catalog as a single JSON document: name, description, license, homepage + the 20 dataset entries with slug, recordCount, refresh date, sample/full URLs, related-slug list.

Safe to poll daily. ~21 KB.

curl -s https://futdevpro.github.io/niche-datasets-free/datasets.json | jq '.datasets | length'
# → 20

GET /{slug}-meta.json

Metadata for one dataset. Returns a single dataset's metadata object — same shape as the entries inside /datasets.json but addressable on its own URL.

Useful for thin clients that only care about one or two datasets.

curl -s https://futdevpro.github.io/niche-datasets-free/ai-models-pricing-meta.json | jq .recordCount
# → "800+"

Valid slugs (20): ai-agents, ai-models-pricing, ai-prompts, ai-tools, cybersecurity-tools, design-resources, developer-tools, homebrew-packages, huggingface-datasets, huggingface-models, llmops-and-eval, mcp-servers, no-code-lowcode, npm-packages, open-source-alternatives, platform-engineering, public-apis, self-hosted-software, vector-db-and-rag, vscode-extensions.

GET /feed.xml

RSS 2.0 refresh feed. One entry per dataset, ordered by lastRefreshed. Subscribe in your feed reader to be notified when any dataset re-pulls.

Auto-generate a client

The spec is OpenAPI 3.1, so any codegen that supports the 3.1 schema works:

# Python
npx @openapitools/openapi-generator-cli generate \
  -i https://futdevpro.github.io/niche-datasets-free/openapi.json \
  -g python \
  -o ./niche-datasets-client

# TypeScript / fetch
npx openapi-typescript \
  https://futdevpro.github.io/niche-datasets-free/openapi.json \
  -o niche-datasets.d.ts

Caching, freshness, rate limit

ConcernBehavior
CacheGitHub Pages CDN; default Cache-Control: max-age=600. Responses are freshness-stamped via the embedded lastRefreshed field — trust that over the HTTP cache header.
Rate limitNone on Pages itself. GitHub may rate-limit raw.githubusercontent.com on the sample URLs at extreme volume (~10k req/min from one IP) — back off with jitter.
Refresh cadencePer-source. The fastest-moving (npm, HuggingFace) get nightly-to-weekly pulls. The slowest (awesome-list mirrors) get monthly. See 13-day refresh report for the per-source churn numbers that drive this cadence.
AuthNone on the sample tier. Full data ZIPs are gated on Gumroad purchase.

Schema versioning

Every JSON response includes a schemaVersion field. Breaking changes will bump it; additive changes will not. Clients should ignore unknown fields (forward-compat).

For agents specifically

This is a free tier — designed for inspection, prototyping, and low-volume agent enumeration. If you need higher-volume access, per-call billing, or full-dataset live endpoints (not just samples), the project is currently in distribution-model planning. Watch the GitHub repo for updates, or open an issue describing your use case.
