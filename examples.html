Python · pandas

Load the CSV directly from the GitHub raw URL into a DataFrame. Works for all 20 datasets — change the slug in the URL.

import pandas as pd

df = pd.read_csv("https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.csv")
print(df.shape)            # (20, N) — N varies per dataset
print(df.columns.tolist())
print(df.head(3))

Python · stdlib json

No pandas dependency — pure stdlib via urllib + json. Returns a list of dicts you can filter directly.

import json, urllib.request

with urllib.request.urlopen("https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.json") as r:
    records = json.load(r)

# Filter: only open-source packages with an API
oss_with_api = [r for r in records if r.get("pricing") == "open-source" and r.get("hasApi")]
print(len(oss_with_api), "of", len(records))

JavaScript · fetch (browser or Node 18+)

Native fetch in any modern browser or Node 18+. No dependencies.

const url = "https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.json";
const records = await fetch(url).then(r => r.json());

console.log(records.length, "records");
console.log(records[0]);

// Group by category
const byCategory = Object.groupBy(records, r => r.category);
console.log(Object.keys(byCategory));

jq CLI

Pipe curl into jq. Great for one-off filtering or pre-pipeline data exploration without a script.

# All names + categories
curl -s https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.json | jq -r '.[] | "\(.name)\t\(.category)"'

# Just the ai-and-ml packages
curl -s https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.json | jq '.[] | select(.category == "ai-and-ml")'

# Count by pricing bucket
curl -s https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.json | jq -r '[.[].pricing] | group_by(.) | map({key: .[0], count: length})'

Pandas-style filter + join (npm + Homebrew supply-chain audit)

Real use-case: cross-reference two datasets. Load both, join on a key, surface risk.

import pandas as pd

npm = pd.read_csv("https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.csv")
brew = pd.read_csv("https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/homebrew-packages-sample.csv")

# Packages that appear in BOTH ecosystems (rare — usually a CLI tool with both an npm wrapper
# and a Homebrew formula). Useful for supply-chain audits: a package that's distributed
# through multiple package managers has more attack surface.
overlap = pd.merge(npm[['name']], brew[['name']], on='name')
print(f"{len(overlap)} packages in both registries")
print(overlap.head())

DuckDB · query JSON/CSV remote URL

DuckDB reads CSV + JSON straight from HTTPS, no download. Perfect for ad-hoc SQL on the catalog.

ORDER BY n DESC;

Semantic enum tier filters (Python · pandas)

Same buyer-question, in Python. Loads from the sample URL straight into pandas, filters by semantic enum tier fields with one boolean-mask combo per use case.

import pandas as pd

BASE = "https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main"

# AI Models Pricing — cheap chat models with production-grade uptime
pricing = pd.read_json(f"{BASE}/ai-models-pricing-sample.json")
cheap_chat = pricing[
    (pricing["useCaseTier"] == "chat") &
    (pricing["costTierAbsolute"] == "cheap") &
    (pricing["uptimeTier"] == "excellent")
]
print(cheap_chat[["name", "provider", "pricingPromptPerMillionUsd", "contextLength"]])

# HuggingFace Models — embedding models only, permissively licensed
hf = pd.read_json(f"{BASE}/huggingface-models-sample.json")
emb_perm = hf[(hf["useCaseTier"] == "embedding") & (hf["licenseTier"] == "permissive")]
print(emb_perm[["modelId", "downloads", "license"]])

# npm Packages — popular, safe, fresh
npm = pd.read_json(f"{BASE}/npm-packages-sample.json")
trusted = npm[
    (npm["popularityTier"] == "top25pct") &
    (npm["supplyChainRisk"] == "low") &
    (npm["recencyTier"] == "fresh")
]
print(trusted[["name", "weeklyDownloads", "lastPublished"]])

Semantic enum tier filters (jq)

Filter by derived enum tier fields — same enum names work across datasets where the buyer-question is the same. Full reference at /tiers.html, methodology at /blog-semantic-enum-tiers.html.

curl -s https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/ai-models-pricing-sample.json \

# HuggingFace Models — embedding models only, permissively licensed
curl -s https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/huggingface-models-sample.json \
  | jq '.[] | select(.useCaseTier == "embedding" and .licenseTier == "permissive")
        | {modelId, downloads, license}'

# npm Packages — popular (top-25%), no supply-chain risk, recently published
curl -s https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.json \
  | jq '.[] | select(.popularityTier == "top25pct" and .supplyChainRisk == "low" and .recencyTier == "fresh")

# Cybersecurity Tools — OSS only, network-targeting
curl -s https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/cybersecurity-tools-sample.json \

        | {name, description}'

Semantic enum tier filters (Node.js · fetch)

JavaScript / Node.js (18+) with native fetch. Same buyer-questions as the jq + pandas snippets, in JS. Useful for agent-builders, API gateways, frontend filtering UIs.

const BASE = 'https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main';

// AI Models Pricing — cheap chat models with production-grade uptime
const pricing = await (await fetch(`${BASE}/ai-models-pricing-sample.json`)).json();
const cheapChat = pricing.filter(r =>
  r.useCaseTier === 'chat' &&
  r.costTierAbsolute === 'cheap' &&
  r.uptimeTier === 'excellent'
);
console.log(cheapChat.map(r => ({
  name: r.name, provider: r.provider,
  pricePerM: r.pricingPromptPerMillionUsd, ctx: r.contextLength,
})));

// HuggingFace Models — embedding models only, permissively licensed
const hf = await (await fetch(`${BASE}/huggingface-models-sample.json`)).json();
const embPerm = hf.filter(r =>
  r.useCaseTier === 'embedding' && r.licenseTier === 'permissive'
);
console.log(embPerm.map(({ modelId, downloads, license }) => ({ modelId, downloads, license })));

// npm Packages — popular, safe, fresh
const npm = await (await fetch(`${BASE}/npm-packages-sample.json`)).json();
const trusted = npm.filter(r =>
  r.popularityTier === 'top25pct' &&
  r.supplyChainRisk === 'low' &&
  r.recencyTier === 'fresh'
);
console.log(trusted.map(({ name, weeklyDownloads, lastPublished }) => ({ name, weeklyDownloads, lastPublished })));

Semantic enum tier filters (DuckDB · SQL)

Same buyer-questions, in SQL via DuckDB reading the JSON straight from HTTPS. Useful when you already have a DuckDB warehouse + want to join the niche-datasets samples against your own tables.

-- in duckdb shell, duckdb-python, or duckdb-wasm
INSTALL httpfs; LOAD httpfs;

-- AI Models Pricing — cheap chat models with production-grade uptime
SELECT name, provider, pricingPromptPerMillionUsd, contextLength
FROM read_json_auto('https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/ai-models-pricing-sample.json')
WHERE useCaseTier = 'chat'
  AND costTierAbsolute = 'cheap'
  AND uptimeTier = 'excellent'
ORDER BY pricingPromptPerMillionUsd ASC;

-- HuggingFace Models — embedding models only, permissively licensed
SELECT modelId, downloads, license
FROM read_json_auto('https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/huggingface-models-sample.json')
WHERE useCaseTier = 'embedding'
  AND licenseTier = 'permissive'
ORDER BY downloads DESC;

-- npm Packages — popular, safe, fresh
SELECT name, weeklyDownloads, lastPublished
FROM read_json_auto('https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/npm-packages-sample.json')
WHERE popularityTier = 'top25pct'
  AND supplyChainRisk = 'low'
  AND recencyTier = 'fresh'
ORDER BY weeklyDownloads DESC;

Bundle vs single-SKU price comparison (jq)

Pick a target use case (e.g. 'agent registry'), enumerate the datasets you need, sum their single-SKU prices, compare against the bundle that includes them. Decision-support for the buyer's guide use-case mapping.

# Use case: build an agent / MCP registry.
# Needed datasets: mcp-servers + ai-agents + ai-tools.
# Compare per-dataset sum vs Platform Builder Pack (which includes platform + dev tools).

curl -s https://futdevpro.github.io/niche-datasets-free/datasets.json \
  | jq '
      .datasets
      | map(select(.slug | IN("mcp-servers", "ai-agents", "ai-tools")))
      | {
          single_skus_total: (map(.fullDatasetPriceUsd) | add),
          single_skus:       map({slug, price: .fullDatasetPriceUsd}),
          bundle_options:    (.[0].bundles // [])
        }'

Change-detection between refreshes (Python · liveRecordCount)

Poll /datasets.json, compare liveRecordCount per dataset against a saved snapshot, alert when anything moves. Use this if you mirror our catalog into your own pipeline and want a 'pull only what changed' loop. Documented in /api.html under 'Two record-count fields'.

import json, urllib.request, pathlib

URL = 'https://futdevpro.github.io/niche-datasets-free/datasets.json'
SNAPSHOT = pathlib.Path('niche-datasets-snapshot.json')

# Fetch current catalog
with urllib.request.urlopen(URL) as r:
    current = {d['slug']: d.get('liveRecordCount') for d in json.load(r)['datasets']}

# Load previous snapshot (or treat first run as 'everything changed')
prev = json.loads(SNAPSHOT.read_text()) if SNAPSHOT.exists() else {}

# Diff
changed = {
    slug: (prev.get(slug), n) for slug, n in current.items()
    if prev.get(slug) != n and n is not None
}

if changed:
    print(f'{len(changed)} datasets changed since last poll:')
    for slug, (was, now) in sorted(changed.items()):
        delta = '+new' if was is None else f'{now - was:+d}'
        print(f'  {slug}: {was} -> {now} ({delta})')
else:
    print('No changes since last poll.')

# Save current as next snapshot
SNAPSHOT.write_text(json.dumps(current, indent=2))

Cross-leverage discovery (jq · which tools span the most datasets)

/cross-dataset-overlap.json ships every URL that appears in 3+ datasets, sorted by appearance count. Use it for market-mapping ("which tools cover the most developer niches?"), or for cross-sell discovery (if a tool spans 4 datasets, buyers of any of those 4 are likely interested in the other 3). Schema: CrossDatasetOverlap in OpenAPI 1.3.0.

# Top 5 tools by cross-dataset span:
curl -s https://futdevpro.github.io/niche-datasets-free/cross-dataset-overlap.json \
  | jq '.topByCount[:5] | .[] | {count, url, datasets: .appearsIn}'

# Which datasets does a specific URL appear in?
curl -s https://futdevpro.github.io/niche-datasets-free/cross-dataset-overlap.json \
  | jq '.topByCount[] | select(.url | contains("qdrant")) | .appearsIn'

# All URLs spanning 4+ datasets (the highest-leverage tools):
curl -s https://futdevpro.github.io/niche-datasets-free/cross-dataset-overlap.json \
  | jq '.topByCount[] | select(.count >= 4) | .url'

Data quality audit (jq · verify zero section-header noise)

The crawler strips two common awesome-list noise patterns at extraction time: (a) year-prefix section-headers captured as list items (e.g. "2021 Result", "2023 Competition"), and (b) self-section pointers where a github.com/<owner>/<repo>#anchor URL alnum-matches the record name. The snippet below re-runs both checks against any sample file — for any clean sample, both counts return 0. Use this as a one-shot quality probe before signing off on a dataset for your pipeline. Methodology: FAQ Q26.

# Year-prefix section-header noise (should be 0):
curl -sS https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/vector-db-and-rag-sample.json \
  | jq '[.[] | select(.name | test("^(19|20)[0-9]{2}\\s+[A-Z][a-z]+$"))] | length'

# Self-section pointer noise (should be 0):
curl -sS https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/self-hosted-software-sample.json \
  | jq '[.[] | select(
      (.url | test("^https?://github\\.com/[^/]+/[^/#]+#")) and
      ((.name | ascii_downcase | gsub("[^a-z0-9]"; "")) ==
       (.url | capture("#(?<a>.+)$").a | ascii_downcase | gsub("[^a-z0-9]"; "")))
    )] | length'

# Sweep all 20 samples for either pattern in one pass:
for slug in homebrew-packages npm-packages vscode-extensions huggingface-models \
            mcp-servers ai-tools cybersecurity-tools huggingface-datasets \
            public-apis self-hosted-software design-resources ai-agents \
            ai-prompts developer-tools open-source-alternatives ai-models-pricing \
            no-code-lowcode llmops-and-eval platform-engineering vector-db-and-rag; do
  n=$(curl -sS "https://raw.githubusercontent.com/futdevpro/niche-datasets-free/main/${slug}-sample.json" \
      | jq '[.[] | select(.name | test("^(19|20)[0-9]{2}\\s+[A-Z][a-z]+$"))] | length')
  printf "%-30s year-prefix noise: %s\n" "$slug" "$n"
done

Schema reference

All 20 datasets share a base schema: name, url, description, category, pricing (free / freemium / paid / open-source), hasApi, tags. Each dataset adds first-class fields specific to its niche — see the per-dataset detail pages (e.g. npm-packages) for the full reference, or the repo README §Format.
