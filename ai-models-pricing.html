AI Models & Providers Pricing Matrix

800+ records · Free sample (20 records, JSON + CSV) · Full dataset $11 · Sample refreshed 2026-05-13

Catalog of 800+ AI model endpoints with per-token pricing (prompt + completion), context length, modality, uptime, across providers and quantizations.

What you'd use this for

  1. AI cost-modeling spreadsheet for a launch decision — PM about to ship an AI feature wants to estimate monthly inference cost. Plug expected daily token volume into the dataset → instantly see the cheapest viable provider per model + the cost spread (5-15× between providers for the same model). Saves a week of manual provider-page scraping.
  2. Provider-rotation logic for production reliability — Production app needs failover when a provider goes down. The dataset's uptimeLast30dPct + pricingPromptPerMillionUsd per endpoint enables a programmatic "primary + 2 fallbacks per model" routing config. Pair high-uptime expensive primary with cheap-secondary fallbacks — your mean cost stays low, your tail latency stays bounded.
  3. Migrate from OpenAI direct → cheaper provider for the same model — The matrix surfaces that the same Llama 3.3 70B is on 15+ providers, with the cheapest at 1/15th of OpenAI's equivalent price. CFO-friendly cost-cut without changing the model.
  4. Compliance-driven provider selection (data-residency, vendor-list) — The dataset includes providerHeadquarters + providerDatacenters. EU-data-residency requirement → filter for HQ in EU + datacenters in EU. SOC2-vendor-list requirement → cross-reference with your approved vendor list. Selection becomes a 1-line filter instead of a procurement spreadsheet.
  5. Newsletter / market-analysis content — A monthly "LLM pricing recap" newsletter needs source data. The dataset provides the entire matrix; the newsletter writer just narrates the deltas (new providers added, top price-drops, category trends). Refresh-monthly cadence aligns perfectly with newsletter cadence.
  6. Quantization-vs-quality tradeoff dashboard — ML engineering team wants to know "if we drop from bf16 to fp8 on Llama 3.3, how much do we save?". Dataset surfaces both quantization tiers per provider with their pricing → instant savings calculation. Answer: usually 30-50% cost reduction.
  7. Capability-driven model picker for an agent framework — Agent framework needs to find "the cheapest model that supports tools AND structured_outputs AND has 128K+ context". Dataset's supportedParameters + contextLength + pricing in one schema → the picker is a 5-line query.
