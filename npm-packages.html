npm Packages Directory

6,000+ records · Free sample (20 records, JSON + CSV) · Full dataset $11 · Sample refreshed 2026-05-13

Catalog of 6,000+ top JavaScript/TypeScript npm packages with weekly/monthly downloads, dependents count, npm score (popularity/quality/maintenance), and repository.

What you'd use this for

  1. Supply-chain risk audit — Security team needs to flag risky deps. For every package in your package.json, look up maintenanceScore from the dataset. Flag anything < 0.5 — packages becoming unmaintained are supply-chain risks. The dataset has this for ~10% of top-6,000 packages.
  2. Dependency health dashboard — Build an internal dashboard showing your project's dependency stack with maintenance / popularity / quality scores. Cross-reference lastPublished to flag deps that haven't shipped in > 18 months. Saves manual "is this still maintained?" research per quarter.
  3. Library selection for a new feature — PM picks "we need to add date manipulation to the app". Filter dataset by category=utility-library + description matches "date" → date-fns, dayjs, luxon, moment surface with their npm scores. Picker is a 2-minute decision, not a half-day research task.
  4. Internal "approved packages" catalog — Engineering manager wants a vetted list of allowed npm packages per category (HTTP clients, ORMs, testing). Filter dataset by category + npmScore > 0.7 + license=MIT|Apache-2.0 → ship the list as an internal Confluence page. New devs pick from the list instead of yolo-installing whatever GitHub recommends.
  5. AI-and-ML JS SDK landscape research — You're building an AI feature in a JS app. Filter category=ai-and-ml → openai, anthropic, langchain, llamaindex, transformers.js, ollama-js all surface. Compare by weeklyDownloads to see which SDK has the strongest community.
  6. Newsletter / blog content sourcing — A weekly "underrated npm packages you should know" newsletter. Filter by category + filter for packages with weeklyDownloads between 10K-100K (high enough to be vetted, low enough to be unknown). Each becomes a tutorial post.
  7. Bootstrap script generator for a new project — You scaffold new JS/TS projects regularly. For each project type (frontend / backend / CLI / monorepo), the dataset's category buckets give you the canonical starter packages. Build a CLI npx my-scaffold --type=cli that pulls top-N from cli-and-devops category → pre-installs commander, yargs, chalk, ora, prompts.
