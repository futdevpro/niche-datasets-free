Catalog of 2,600+ most-downloaded HuggingFace datasets with downloads, likes, license, task categories, and language.

What you'd use this for

  1. Fine-tuning starter dataset selection — Researcher fine-tuning a base model needs SFT data. Filter category=instruct-and-chat (101 records) → curated list of every viable SFT dataset on HF Hub. Pick by downloads (vetted) + language + size_categories. Skip the 250K-dataset Hub UI scroll.
  2. RLHF / preference-data hunting — DPO/RLHF training needs preference data. Filter category=preference-and-rlhf (10 records) → the entire viable open-source RLHF data set in one query. Anthropic HH-RLHF, ultrafeedback, nectar all surfaced.
  3. Eval-harness setup for a custom benchmark suite — Custom eval needs benchmark datasets. Filter category=benchmark-and-eval (494 records) → MMLU, GSM8K, HumanEval, BigBench, ARC-Challenge, HellaSwag, TruthfulQA all surfaced. Build the eval harness from the categorized subset instead of scrolling.
  4. Multimodal training-data scout — Building a vision-language model. Filter category=multimodal OR modality=image (cross-cut) → 600+ image+text datasets including CLIP-training data, VQA datasets, image-captioning corpora. The dataset's structured task_categories field surfaces VQA / DocVQA / image-to-text variants separately.
  5. License-compliance audit for an existing training pipeline — ML team has a multi-dataset training mix. Each dataset's license is buried in HF tags as license:apache-2.0 strings. Cross-reference the dataset's license field with your training-mix manifest → flag any non-commercial-only datasets that would block commercial release of the trained model.
  6. Translation-model training-data picker — Building a translation model. Filter category=translation → 47 datasets including WMT, OPUS, Tatoeba, NLLB, FLORES. Sort by language + size_categories → pick the right corpus for your language pair without scrolling through pretraining datasets.
  7. Cohort analysis: what training data is the community converging on? — lastModified + downloads enables cohort tracking. Filter to "datasets modified in last 6 months" + sort by downloads → which dataset is the community currently most actively maintaining? Often signals where the next wave of fine-tunes will originate.
